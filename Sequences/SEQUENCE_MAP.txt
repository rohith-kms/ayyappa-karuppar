SEQUENCE_MAP.txt — THE DETERMINISTIC MASTER LIST  (do not re-segment)
==============================================================================
The reel is split into 22 sequences, one per script beat. The engine fills these
folders IN ORDER and never re-numbers or re-splits them. This map is the single
source of truth for "how many sequences" and "what each one is."

COLUMNS:
  SEQ        folder name (Sequence_NN)
  TIMECODE   from Script.txt
  BEAT       short label
  START      FRESH_START = this is a hard cut / new scene, generate its own
                           Start-image-prompt.txt
             CONTINUES   = inherit the PREVIOUS sequence's end.png as the start
                           frame (NO Start-image-prompt.txt is written)
  CHARACTERS who appears (which CHARACTER_BIBLE blocks to embed)

NOTE on START rule: Sequence_01 ALWAYS gets both Start + End. After that, only
FRESH_START sequences get a Start-image-prompt. This is the upgrade to the
"only Seq 1 has a start" idea — it keeps continuity where the picture is truly
continuous, and gives a fresh frame at every hard cut (a trailer has many).
