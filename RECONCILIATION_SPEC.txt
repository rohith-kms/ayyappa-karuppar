==============================================================================
RECONCILIATION_SPEC.txt
Mechanical spec for syncing BIBLE LOCKED BLOCKS to the approved reference plates.
==============================================================================

>>> WHO RUNS THIS: a multimodal cheap model (e.g., Claude Haiku, GPT-4o-mini).
    The producer attaches the reference plates listed below AND points the
    model at this spec + CHARACTER_BIBLE.txt + ENVIRONMENT_BIBLE.txt.

>>> TRIGGER PHRASE: "Read RECONCILIATION_SPEC.txt and RUN"
    Optional: "Read RECONCILIATION_SPEC.txt and RUN <subject>" — limit to one
    subject (e.g., "RUN Karuppar" or "RUN Bengal Tiger").

>>> WHEN: after reference plates are saved into their folders, or after any
    plate regeneration.

>>> WHAT CHANGES: only the LOCKED BLOCK paragraphs inside CHARACTER_BIBLE.txt
    and ENVIRONMENT_BIBLE.txt. NEVER touches anything else.

THE PROCEDURE  (do this for EACH subject in the map)
==============================================================================
STEP 1 — LOAD THE TEXT.
  Open the matching BIBLE file. Find the heading per the Subject Map. Extract
  the current LOCKED BLOCK paragraph(s) that follow.

STEP 2 — LOOK AT THE PLATE.
  Carefully observe the image. For multi-panel plates (army character systems),
  focus on the specific panel called out in the Subject Map. For
  characters with multiple states (Shiva, Karuppar), each state has its own
  plate and its own LOCKED BLOCK to reconcile.

STEP 3 — COMPARE.
  Identify every detail where the IMAGE differs from the TEXT.

STEP 4 — WRITE THE REPLACEMENT.
  Produce an updated paragraph in the same structure and voice. Apply the
  KEEP/CHANGE rules below.

STEP 5 — OUTPUT.
  See OUTPUT FORMAT.

==============================================================================
KEEP / CHANGE RULES
==============================================================================

KEEP (never modify):
  - Cultural-anchor language ("Tamil Hindu mythological", "Sanatan Dharma",
    "South Indian", "Dravidian", "Sabarimala tradition", "village-deity
    tradition", "Karuppar tradition", etc.)
  - Reverence cues for Shiva and Ayyappa
  - Functional identity markers (Karuppar's cheroot, chains, aruval; Mahishi's
    buffalo head, cleaver, lion-shield; Shiva's trident, crescent moon, Ganga;
    Black Dog's amber-brown eyes; White Horse's red caparison)
  - Identity-Lock items (paragraph length, voice, comma-chained descriptive)
  - Archetype rank labels and heights (these are taxonomic, not visual)

CHANGE (sync to what the plate actually shows):
  - Exact skin / hide / coat tone wording
  - Exact eye color and luminosity
  - Specific garment colors, patterns, layering
  - Armor materials, ornaments, jewelry
  - Headgear style, hair, horn shape/length
  - Facial-marking placement, color, count
  - Body proportions if plate clearly disagrees with spec
  - Background / sky / terrain palette (environments)
  - Any clearly visible detail the locked block did not mention

KARUPPAR SPECIAL RULE:
  The team will add the cheroot to the plate POST-generation. So even if the
  current plate version you see lacks the cheroot, KEEP the cheroot
  description in the locked block. The cheroot is canonical.

SHIVA SPECIAL RULE:
  Shiva has TWO locked blocks (meditative + wrathful). Reconcile EACH against
  ITS OWN plate. Do not merge.

ARCHETYPE SPECIAL RULE:
  When reconciling an archetype block, focus only on the labeled panel for
  that archetype in the character system plate. Ignore the other panels.

NEVER add to the locked block:
  - Camera framing, lighting direction, lens / depth-of-field language
  - Compositional notes
  - Cinematic descriptors (those live in STYLE token)
  - Negatives (those live in NEGATIVES line below the block)
  - References to specific sequences

NEVER touch (outside the LOCKED BLOCK):
  - The NEGATIVES (specific) line
  - The REFERENCE line
  - The IDENTITY LOCKS list
  - The WEAPONS / MOUNT / COMPANION lines
  - The cultural-anchor preamble at the top of the BIBLE file
  - Any other subject's locked block

==============================================================================
HARD RULES
==============================================================================
  - Output text only — never write files directly. Producer pastes manually.
  - If a plate is missing, SKIP that subject and note "PLATE MISSING".
  - If a plate clearly does not match the subject, flag "PLATE MISMATCH" and
    do not produce a replacement.
  - Never invent details not visible on the plate.
  - Never remove cultural-anchor or reverence language.
  - "RUN <subject>" processes ONLY that subject.
