==============================================================================
CHARACTER_BIBLE.txt — LOCKED DESCRIPTORS  (the key to visual consistency)
==============================================================================
WHY THIS EXISTS: AI image tools "forget" what a character looks like between
generations. To keep every character identical across all 22 sequences, EVERY
image prompt must embed the exact LOCKED BLOCK below for each character that
appears. Do not paraphrase these blocks — copy them verbatim. The reference
plates live in /Character-Sheets/.

ORGANIZATION (this file has 5 sections):
  §1  HEROES                    — Karuppar, Ayyappa, Mahishi, Shiva
  §2  DIVINE CREATURES          — Bengal Tiger, White Horse, Black Dog
  §3  ARMY OF DHARMA            — 4 archetypes (Ayyappa's forces)
  §4  ARMY OF THE ASURAS        — 5 archetypes (Mahishi's legions)
  §5  WEAPONS & STANDARDS       — named weapon catalog

CULTURAL ANCHOR REMINDER:
  Every prompt must also include the GLOBAL_CULTURAL_TOKEN from CONFIG.txt.
  Without it, the model defaults to Western imagery (Vikings, knights, generic
  D&D demons). The LOCKED BLOCKS below already use Tamil-Hindu vocabulary —
  do not "translate" terms like vel, aruval, dhoti, veshti, vibhuti, tilak,
  jata, rudraksha, talwar, khanda into Western equivalents.

ANTI-CLONING REMINDER (NEW):
  Any beat that shows ≥3 figures of the same archetype must also include the
  GLOBAL_ANTI_CLONING_TOKEN from CONFIG.txt. Crowds of identical warriors are
  the second-biggest visual failure mode after Western drift.

Each block has:
  LOCKED BLOCK         = paste this verbatim into prompts
  NEGATIVES (specific) = per-subject negative overlay, added on top of the
                         global NEGATIVE_PROMPT in CONFIG.txt
  REFERENCE            = which plate to attach as image reference
  IDENTITY LOCKS       = visual features that must NEVER change between shots
                         (face, eyes, signature ornaments, etc.)
