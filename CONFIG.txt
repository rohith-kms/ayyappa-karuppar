==============================================================================
CONFIG.txt — GLOBAL SETTINGS FOR ALL GENERATED PROMPTS
==============================================================================
Change a value here and EVERY future prompt the engine writes will use it.
The producer should confirm the three [CONFIRM] values below before the image
team starts. Everything else has a sensible default.

------------------------------------------------------------------------------
[CONFIRM] ASPECT_RATIO        = 9:16   (vertical reel for Instagram / YT Shorts)
          Alternatives: 16:9 (landscape), 2.39:1 (cinematic scope).
          If you change this, also change the MJ flag in IMAGE_PARAMS below.
          NOTE: Reference plates use --ar 1:1 — see "REFERENCE PLATE OVERRIDES"
          below. Only the in-reel sequence frames use 9:16.

[CONFIRM] IMAGE_TOOL          = Midjourney v6 / Flux  (prompts are written in
          tool-agnostic natural language so they also work in SDXL, DALL-E, etc.
          The IMAGE_PARAMS line is the only tool-specific part.)

[CONFIRM] VIDEO_TOOL          = Runway Gen-3 / Kling / Luma  (animation prompts
          are natural-language motion descriptions that work across all three.)

==============================================================================
GLOBAL_CULTURAL_TOKEN  (MANDATORY — prepended/woven into EVERY image prompt)
==============================================================================
Why this exists: image models default to Western/European martial imagery
(Vikings, knights, Roman legionaries, Tolkien orcs) because their training data
is Western-heavy. Without an explicit cultural anchor, our Tamil-Hindu warriors
render as crusaders and our asuras render as generic D&D demons. This token
forces the model toward the correct iconography.

  "ancient Tamil Hindu mythological warriors, Sanatan Dharma iconography, South
  Indian / Dravidian martial culture, Chola-era warrior styling, bronze and
  brass weaponry of the Indian subcontinent, dhoti / veshti / angavastram
  garments, temple-fresco and Tanjore-painting aesthetic, dark South Indian skin
  tones, Tamil epic warfare"

The producer's engine (Sequences/Generate-sequences.txt) auto-injects this into
every prompt it writes. Reference-prompt.txt files include it explicitly.
Backpatching the cultural token into older prompts is covered by
SEQUENCE_BACKPATCH_SPEC.txt at the repo root.

==============================================================================
GLOBAL_ANTI_CLONING_TOKEN  (MANDATORY for any beat with >=3 same-archetype figures)
==============================================================================
Why this exists: image models default to repeating identical figures in crowd
shots — every Gana looks like the same Gana, every asura wears the same face.
The Final-images army taxonomies explicitly show variation WITHIN each rank
(different ages, builds, beards, hair, scars, weapon loadouts). This token
forces the model to honor that variation.

Inject this into the prompt whenever a frame shows 3 or more figures of the
same archetype (most battlefield wides, all aerial shots, every crowd close).

  POSITIVE FORM:
    "each warrior is a distinct individual — varied facial structures, varied
    skin-tone gradients within the South Indian range, varied builds (lean /
    heavy / wiry), varied ages (young / scarred-veteran / grey-bearded
    elder), varied hair (loose / topknot / braided / shaved / balding),
    varied beard length and grey, varied battle wear and scars, unique
    ornament and weapon-loadout per soldier, no two warriors share the
    same face or stance, natural physiognomic variety within each rank"

  NEGATIVE ADDITIONS (append to NEGATIVE_PROMPT when this token is in use):
    "identical faces, cloned warriors, duplicated soldiers, copy-paste
    army, regiment of clones, same face on every character, photocopied
    warriors, repeating mannequin figures, uniform identical fighters,
    twin army, NPC army"

The engine (Sequences/Generate-sequences.txt) auto-injects this whenever the
beat's character/archetype count >=3 of the same type. The backpatch spec
(SEQUENCE_BACKPATCH_SPEC.txt) lists the per-beat trigger explicitly.

ASURA-SPECIFIC ANTI-CLONING NOTE:
  The Beast-Blooded Shock Troops (Asura Army Archetype B) are the highest
  cloning risk because the beast-head is a strong visual hook. Vary the
  beast-head type across figures — some lion-headed, some wolf-headed, some
  hyena-headed, some boar-tusked. Never repeat the same beast-head twice in
  adjacent figures.

DHARMA-SPECIFIC ANTI-CLONING NOTE:
  Dharma Infantry (Archetype A) appears in the largest numbers and is most
  prone to cloning. Vary age across the rank — include young recruits,
  middle-aged hardened, and grey-bearded elders.

GLOBAL_STYLE_TOKEN  (prepended/woven into every image prompt):
  "dark painterly cinematic fantasy, hyper-detailed realism, dramatic
  chiaroscuro storm lighting, volumetric god-rays, fine film grain, epic
  mythological scale, photoreal skin and metal textures, moody crimson-charcoal
  palette, ultra sharp focus, 8k, cinematic color grade"

IMAGE_PARAMS  (append to Midjourney prompts; adapt for other tools):
  "--ar 9:16 --style raw --stylize 250 --quality 2"

REFERENCE PLATE OVERRIDES  (used by Reference-prompt.txt files ONLY):
  PARAMS:  --ar 1:1 --style raw --stylize 200 --quality 2
  COMPOSITION: single isolated subject / clean establishing wide; neutral grey
               or solid backdrop; even diffuse lighting; no other characters
               (unless the plate is intentionally a group plate); full body /
               full subject visible; front + 3/4 view where applicable.
  REASON: a reference plate's job is to lock VISUAL IDENTITY, not to be a
          cinematic frame. The locked block embedded in every later 9:16
          sequence prompt must match what the plate shows.

PALETTE BY MOVEMENT  (the engine applies the right one per sequence):
  MOVEMENT I  (Drowning) : near-monochrome, bruised grey + charcoal, sick orange
                           firelight only. Drained, hopeless.
  MOVEMENT II (Shiva)    : cool blue-white, cosmic, clean, un-bloodied. Calm.
  MOVEMENT III(Arrival)  : storm-dark with one shaft of pale gold light; first
                           crimson + the white horse hit hard here.
  MOVEMENT IV (Carnage)  : full crimson + black + ash-grey, hot embers, blood
                           spray, lightning. Maximum saturation of red.

CONTINUITY NOTE:
  Color must shift gradually WITHIN a sequence only as the script dictates. Do
  not introduce Movement-III color into a Movement-I prompt, etc.

PROMPT BUDGET  (if your tool truncates long prompts, trim in this order):
  1. Style token  (least essential — replaceable with shorter "cinematic")
  2. Image params (tool can usually be set with UI controls)
  3. Locked block (last resort — drift risk increases)
  NEVER trim: cultural anchor or negative prompt. Those are what stop Western
  drift, and Western drift is the most common failure mode in this project.

OUTPUT FILENAMES (what the teams save INTO each Sequence folder):
  start.png       (image team — only exists if a Start-image-prompt.txt exists)
  end.png         (image team — every sequence)
  animation.mp4   (video team)
  image-notes.txt (image team — record seed, tool, version, any tweaks)
