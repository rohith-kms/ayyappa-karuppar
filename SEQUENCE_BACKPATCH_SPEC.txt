==============================================================================
SEQUENCE_BACKPATCH_SPEC.txt
Mechanical spec for retrofitting EXISTING sequence prompts after global changes.
==============================================================================

>>> WHO RUNS THIS: a cheap model (Haiku, GPT-4o-mini, etc.) operating on the
    repo. The producer feeds it this spec and one sequence folder at a time.
    Output is rewritten prompt files in the same sequence folder.

>>> TRIGGER PHRASE: "Read SEQUENCE_BACKPATCH_SPEC.txt and RUN"
    Variants:
      "Read SEQUENCE_BACKPATCH_SPEC.txt and RUN Sequence_NN"   (one sequence)
      "Read SEQUENCE_BACKPATCH_SPEC.txt and RUN ALL"           (every existing
                                                                sequence folder
                                                                with prompts)
    Default (no variant) = RUN ALL existing sequences with prompt files.

>>> WHEN TO RUN: any time CONFIG.txt's GLOBAL_CULTURAL_TOKEN,
    GLOBAL_ANTI_CLONING_TOKEN, or NEGATIVE_PROMPT changes, or any time
    CHARACTER_BIBLE.txt / ENVIRONMENT_BIBLE.txt is revised.

>>> WHAT CHANGES: only the five mandatory injections below. NEVER touch the
    creative content of the prompt (composition, camera, action, VO, timing).

------------------------------------------------------------------------------
SCOPE — WHICH FILES TO TOUCH
------------------------------------------------------------------------------
INSIDE EACH Sequences/Sequence_NN/ folder, rewrite these files when present:
   Start-image-prompt.txt
   End-image-prompt.txt
   Animation-prompt.txt

Do NOT rewrite: image-notes.txt, the engine config, or anything outside
Sequences/Sequence_NN/.

------------------------------------------------------------------------------
THE FIVE MANDATORY INJECTIONS
------------------------------------------------------------------------------

INJECTION 1 — CULTURAL ANCHOR  (every image and animation prompt)
  Every prompt file must contain a "CULTURAL ANCHOR:" section near the top.
  Paste the current GLOBAL_CULTURAL_TOKEN body from CONFIG.txt verbatim.

INJECTION 2 — EXPANDED NEGATIVE_PROMPT  (every image prompt)
  Replace the NEGATIVE: line with the full current NEGATIVE_PROMPT from
  CONFIG.txt. Do not abbreviate. Then append per-subject negatives from each
  character/archetype's NEGATIVES (specific) line in CHARACTER_BIBLE.txt as
  a SUBJECT-SPECIFIC NEGATIVES block.

INJECTION 3 — ARMY ARCHETYPE SWAP  (per the archetype tables below)
  Replace any generic "asuras", "asura demons", "asura horde", "Ganas",
  "soldiers", or single old-spec asura variant ("Asura Foot-Soldier",
  "Asura Brute", "Asura Commander") with the new ARMY ARCHETYPE vocabulary:

  DHARMA ARMY (Ayyappa's forces) — 4 archetypes:
    A — Dharma Infantry         (5'8"-6'0", frontline, spear+shield+sword)
    B — Veteran Warrior         (6'1"-6'5", battle-hardened, sword/mace+heavy shield)
    C — Commander               (6'4"-6'10", unit leader, ornate gear + standard)
    D — Divine General          (6'10"-7'4", supreme rank, royal armor)

  ASURA ARMY (Mahishi's legions) — 5 archetypes:
    A — Asura Infantry              (5'5"-5'8", expendable horde)
    B — Beast-Blooded Shock Troop   (6'5"-7'0", animal-headed elite vanguard)
    C — Giant Battlefield Breaker   (10'0"-13'0", towering heavy)
    D — Asura Chieftain             (7'0"-8'0", commander of legions)
    E — Mahishi Elite Guard         (8'0"-9'0", Mahishi's personal guard)

  For each archetype present in the beat, embed its full LOCKED BLOCK
  verbatim from CHARACTER_BIBLE.txt §3 or §4.

INJECTION 4 — ENVIRONMENT LOCKED BLOCK  (per the environment table below)
  Embed the matching ENVIRONMENT_BIBLE.txt LOCKED BLOCK verbatim. Add an
  "IMAGE REFERENCES:" line listing the environment master plate.

INJECTION 5 — ANTI-CLONING TOKEN  (NEW — for any beat with >=3 same-archetype
                                    figures; see Anti-Cloning Trigger Table)
  Inject the GLOBAL_ANTI_CLONING_TOKEN positive form near the bottom of the
  positive prompt, AND append the GLOBAL_ANTI_CLONING_TOKEN negative
  additions to the NEGATIVE line. Both from CONFIG.txt verbatim.

------------------------------------------------------------------------------
WHAT NOT TO TOUCH (hard rules)
------------------------------------------------------------------------------
NEVER modify:
  - Camera moves, framing language
  - Timing / duration
  - Action choreography
  - VO lines
  - The header line  [SEQUENCE NN — <beat> — START/END FRAME]
  - The Movement palette line
  - PARAMS line (--ar 9:16 for sequence frames; reference plates use 1:1)
  - Filenames
  - The START flag in SEQUENCE_MAP.txt (FRESH_START vs CONTINUES)
  - Anything in image-notes.txt

NEVER add:
  - New beats
  - New characters not in CHARACTER_BIBLE.txt
  - Movement palette tints from a different movement

NEVER remove:
  - The reverence cues for Shiva and Ayyappa
  - Continuity references (e.g., "carries through from previous end.png")

------------------------------------------------------------------------------
OUTPUT FORMAT
------------------------------------------------------------------------------
For each rewritten file, output:

  ===== <full path to file> =====
  <full new file contents>

After all files in the sequence are rewritten, output a one-line summary:
  "Backpatched Sequence_NN: <N> files updated."

------------------------------------------------------------------------------
SELF-CHECK (run on every file after rewriting)
------------------------------------------------------------------------------
[ ] CULTURAL ANCHOR section present and matches current GLOBAL_CULTURAL_TOKEN.
[ ] If asuras present: correct Asura Army archetype LOCKED BLOCK(s) embedded
    from CHARACTER_BIBLE.txt §4. The old "Asura Foot-Soldier / Brute /
    Commander" wording is NOT present.
[ ] If Ayyappa's army present: correct Dharma Army archetype LOCKED BLOCK(s)
    embedded from CHARACTER_BIBLE.txt §3. The old "Ganas" wording is NOT
    present (use archetype names instead).
[ ] If beat has an environment: matching ENVIRONMENT_BIBLE.txt LOCKED BLOCK
    embedded verbatim.
[ ] Full expanded NEGATIVE_PROMPT from CONFIG.txt present.
[ ] SUBJECT-SPECIFIC NEGATIVES listed for every subject present.
[ ] If beat is in the Anti-Cloning Trigger Table as YES: anti-cloning token
    positive form is embedded AND anti-cloning negative additions are in the
    NEGATIVE line.
[ ] PARAMS line unchanged (--ar 9:16 for sequence frames).
[ ] IMAGE REFERENCES lists every reference plate: hero master sheets,
    archetype reference plates (from Character-Sheets/Dharma-Army/ or
    /Asura-Army/), weapons catalog (if a named weapon appears), environment
    plate(s).
[ ] Camera moves, action, VO, timing, Movement palette UNCHANGED.
[ ] No new beats invented. No content removed.
