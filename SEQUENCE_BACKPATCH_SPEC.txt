==============================================================================
SEQUENCE_BACKPATCH_SPEC.txt
Mechanical spec for retrofitting EXISTING sequence prompts after global changes.
==============================================================================

>>> WHO RUNS THIS: a cheap model (Haiku, GPT-4o-mini, etc.) operating on the
    repo. The producer feeds it this spec and one sequence folder at a time.
    Output is rewritten prompt files in the same sequence folder.

>>> TRIGGER PHRASE: "Read SEQUENCE_BACKPATCH_SPEC.txt and RUN"
    Variants:
      "Read SEQUENCE_BACKPATCH_SPEC.txt and RUN Sequence_NN"   (one sequence)
      "Read SEQUENCE_BACKPATCH_SPEC.txt and RUN ALL"           (every existing
                                                                sequence folder
                                                                with prompts)
    Default (no variant) = RUN ALL existing sequences with prompt files.

>>> WHEN TO RUN: any time CONFIG.txt's GLOBAL_CULTURAL_TOKEN or NEGATIVE_PROMPT
    changes, or any time CHARACTER_BIBLE.txt / ENVIRONMENT_BIBLE.txt is
    revised. Today's first run: backpatch Sequence_01 (only sequence that has
    prompts so far). Future runs apply to whatever sequences exist.

>>> WHAT CHANGES: only the four mandatory injections below. NEVER touch the
    creative content of the prompt (composition, camera, action, VO, timing).

------------------------------------------------------------------------------
SCOPE — WHICH FILES TO TOUCH
------------------------------------------------------------------------------
INSIDE EACH Sequences/Sequence_NN/ folder, rewrite these files when present:
   Start-image-prompt.txt
   End-image-prompt.txt
   Animation-prompt.txt

Do NOT rewrite: image-notes.txt, the engine config, or anything outside
Sequences/Sequence_NN/.

------------------------------------------------------------------------------
THE FOUR MANDATORY INJECTIONS
------------------------------------------------------------------------------

INJECTION 1 — CULTURAL ANCHOR  (every image and animation prompt)
  Every prompt file must contain a "CULTURAL ANCHOR:" section near the top
  (after the header line but before the "subject paragraph"). The literal
  text is the current value of GLOBAL_CULTURAL_TOKEN in CONFIG.txt:

    CULTURAL ANCHOR:
    ancient Tamil Hindu mythological warriors, Sanatan Dharma iconography,
    South Indian / Dravidian martial culture, Chola-era warrior styling,
    bronze and brass weaponry of the Indian subcontinent, dhoti / veshti /
    angavastram garments, temple-fresco and Tanjore-painting aesthetic,
    dark South Indian skin tones, Tamil epic warfare

  If a CULTURAL ANCHOR section already exists, REPLACE its body with the
  current GLOBAL_CULTURAL_TOKEN text. Do not append.

INJECTION 2 — EXPANDED NEGATIVE_PROMPT  (every image prompt; animation
                                          prompts may keep a shorter form)
  In every image prompt file, find the "NEGATIVE:" line (or block) and REPLACE
  it with the full current NEGATIVE_PROMPT from CONFIG.txt. The full list is
  long — paste it verbatim. Do not abbreviate "etc."

  ALSO append per-subject negatives where appropriate: for every character or
  asura variant present in the prompt, append its SUBJECT-SPECIFIC NEGATIVE
  from CHARACTER_BIBLE.txt onto a new line labeled:
    SUBJECT-SPECIFIC NEGATIVES:
    <one line per subject present in this prompt>

INJECTION 3 — ASURA VARIANT SWAP  (per the variant table below)
  Where the existing prompt says "asuras" generically, replace with the
  specific variant(s) from the table below. For crowd shots showing mixed
  variants, list each variant present with its LOCKED BLOCK from
  CHARACTER_BIBLE.txt.

  Keep all narrative wording around the asura mention intact. Only the noun
  phrase changes (and the locked block gets embedded).

INJECTION 4 — ENVIRONMENT LOCKED BLOCK  (per the environment table below)
  Where the existing prompt describes a location, embed the corresponding
  ENVIRONMENT_BIBLE.txt LOCKED BLOCK verbatim. The original prose around the
  location stays — the locked block is ADDED, structurally similar to how
  character locked blocks are embedded.

  Add an "ENVIRONMENT REFERENCE:" line listing the master plate filename
  (e.g., Environment-Sheets/Battlefield/Battlefield_master_plate.png) so the
  image team knows which plate to attach.

------------------------------------------------------------------------------
WHAT NOT TO TOUCH (hard rules)
------------------------------------------------------------------------------
NEVER modify:
  - Camera moves, framing language ("CAMERA:", "PUSH:", "WHIP:", etc.)
  - Timing / duration
  - Action choreography (who does what, in what order)
  - VO lines
  - The header line  [SEQUENCE NN — <beat> — START/END FRAME]
  - The Movement palette line
  - PARAMS line (aspect ratio stays 9:16 for sequence frames; only reference
    plates use 1:1, and those live elsewhere)
  - Filenames (Start-image-prompt.txt / End-image-prompt.txt / Animation-prompt.txt)
  - The START flag in SEQUENCE_MAP.txt (FRESH_START vs CONTINUES)
  - Anything in image-notes.txt

NEVER add:
  - New beats
  - New characters not in CHARACTER_BIBLE.txt
  - Movement palette tints from a different movement

NEVER remove:
  - The reverence cues for Shiva and Ayyappa
  - Continuity references (e.g., "carries through from previous sequence's end.png")

SELF-CHECK (run on every file after rewriting)
------------------------------------------------------------------------------
[ ] CULTURAL ANCHOR section is present and contains the current
    GLOBAL_CULTURAL_TOKEN text.
[ ] If the beat has asuras: the correct variant LOCKED BLOCK(s) per the
    Asura Variant Map are embedded; the generic word "asuras" no longer
    appears as a description (it may still appear as a noun in narrative
    sentences, e.g., "asura demon foot-soldier").
[ ] If the beat has an environment: the ENVIRONMENT LOCKED BLOCK is embedded
    verbatim from ENVIRONMENT_BIBLE.txt.
[ ] The full expanded NEGATIVE_PROMPT from CONFIG.txt is present.
[ ] SUBJECT-SPECIFIC NEGATIVES are listed for every subject present.
[ ] PARAMS line is unchanged (--ar 9:16 for sequence frames).
[ ] IMAGE REFERENCES list every reference plate the prompt needs.
[ ] Camera moves, action, VO, timing, and Movement palette line are
    UNCHANGED from the original.
[ ] No new beats invented. No content removed.
